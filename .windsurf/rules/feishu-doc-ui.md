# 飞书文档 UI 复刻规则

本项目目标是高保真复刻飞书文档富文本编辑体验。AI 在处理飞书文档相关 UI/交互任务时，必须优先遵守本文规则，避免自行脑补功能或样式。

## 核心原则

- **依据优先**：所有 UI、交互、菜单、状态必须以用户提供截图、项目规格文档或飞书真实行为为依据。
- **最小改动**：每次只修复当前明确问题，不做大范围“顺手优化”。
- **禁止脑补**：不允许根据通用编辑器经验自行新增飞书截图中没有的功能、按钮、菜单项或高亮效果。
- **先拆解再实现**：修改复杂交互前，必须先明确目标行为、当前行为、差异点、DOM/CSS/事件分层。
- **不确定就问**：无法确认飞书真实行为时，必须询问用户或只实现确定的最小行为。

## 禁止行为

- 不允许把插入区、选择区、拖拽区混为一个交互区域。
- 不允许点击灰色选择区时触发插入行/列。
- 不允许用临时蓝色整表高亮替代真实的行/列/单元格选区。
- 不允许为了“功能完整”私自增加截图中没有的菜单项。
- 不允许忽略已有样式层级，直接堆叠 `!important` 造成后续难以维护。
- 不允许在未检查现有实现前直接猜测文件或 API。

## 开发前必须确认

处理飞书 UI 复刻任务前，AI 必须确认以下事项：

- **目标组件**：本次修改的是表格、块柄、菜单、工具栏还是其它组件。
- **目标状态**：默认态、hover 态、选中态、菜单打开态、拖拽态分别是什么。
- **交互分区**：哪些区域响应 click，哪些区域响应 hover，哪些区域仅展示。
- **事件边界**：哪些事件必须 `preventDefault`，哪些事件必须 `stopPropagation`。
- **验收标准**：根据截图或规格文档列出可检查项。

## 表格交互强规则

- 顶部灰色轨道是列选择区。
- 左侧灰色轨道是行选择区。
- 行/列边界灰点与蓝色加号是插入区。
- 点击顶部灰色轨道只能选中整列，不得插入列。
- 点击左侧灰色轨道只能选中整行，不得插入行。
- 只有 hover/click 边界灰点或蓝色加号才允许插入行/列。
- 非选中状态下，表格单元格必须保持白底。
- 选中列时：对应顶部轨道段变蓝，对应列单元格显示浅蓝选区。
- 选中行时：对应左侧轨道段变蓝，对应行单元格显示浅蓝选区。
- 打开表格块菜单不代表整张表蓝色选中。

## 菜单规则

- 表格块菜单必须以截图和规格为准。
- 不得私自添加“合并单元格”“背景色”“删除行列”等截图外菜单项，除非用户明确要求。
- 如果用户要求新增功能，必须说明它属于“新增能力”，而不是“飞书截图已有能力”。

## 验证要求

每次修改后至少检查：

- 是否新增了截图外功能。
- 是否存在 hover/click 区域冲突。
- 是否错误触发插入、删除或菜单。
- 是否有 CSS 层级覆盖导致状态不生效。
- 是否破坏已有表格滚动、列宽、块柄或菜单行为。

## 推荐响应方式

向用户汇报时必须简洁说明：

- 改了哪些文件。
- 修复了哪个具体行为。
- 还有哪些未验证项。
- 如果构建或测试被取消，需要明确说明未完成验证。
