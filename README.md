# 飞书文档编辑器复刻项目

> 基于当前项目实现，持续推进飞书文档核心编辑体验、块编辑面板、Slash 菜单、文档管理与协作能力复刻。

## 项目定位

本项目是一个前后端分离的飞书文档编辑器复刻工程。当前重点不是一次性完成飞书全部能力，而是基于已有 TipTap/ProseMirror 编辑器逐步推进：先完成基础文档编辑和飞书风格 UI，再补齐块级操作、真实高级块、评论分享和协同能力。

当前已具备可运行的文档编辑器雏形，核心推进方向是把“看起来像飞书”升级为“交互和数据行为也接近飞书”。

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| 富文本编辑器 | TipTap (基于 ProseMirror) |
| 前端构建 | Vite |
| UI 样式 | Less + 自定义飞书风格样式 |
| 后端框架 | Node.js + Express + TypeScript |
| 数据库 | SQLite (better-sqlite3) |
| API 规范 | RESTful JSON API |

## 当前与飞书的主要差距

- [ ] 文档标题区差距：标题字号、字重、占位符颜色、作者信息、更新时间、分享入口、更多操作入口需要继续贴近飞书。
- [ ] 顶部工具栏差距：按钮尺寸、图标线宽、激活态、分组分割线、下拉菜单位置、禁用态、hover 态仍需统一飞书规范。

## 快速开始

### 安装依赖

```bash
# 安装后端依赖
cd server
npm install

# 安装前端依赖
cd ../client
npm install
```

### 启动开发环境

```bash
# 启动后端服务 (端口 3001)
cd server
npm run dev

# 启动前端开发服务器 (端口 5173)
cd ../client
npm run dev
```

### 访问

打开浏览器访问 http://localhost:5173

## 常用命令

```bash
# 前端构建检查
cd client
npm run build

# 后端测试
cd server
npm test
```

## 当前开发原则

- **先对齐当前实现**：文档中的功能状态必须与代码现状一致，不能提前标记未完成能力。
- **先复刻核心体验**：优先完成块菜单、Slash 菜单、选区工具栏、块操作入口。
- **占位能力要标注清楚**：表格、分栏、同步块、绘图类块目前多数是占位，不等同于飞书真实能力。
- **每次推进一个模块**：先做视觉规格，再做交互逻辑，最后补数据模型和后端 API。
