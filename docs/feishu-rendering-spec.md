# 飞书文档显示与 UI 还原规范

本文档定义本项目“本地显示飞书文档并尽量和飞书一致”的实现目标。后续 UI 修改、导入修复和 Bitable 还原都应优先对齐本文档，而不是只针对单张截图临时修补。

## 目标

本项目的目标不是复用飞书私有前端代码，而是在本地 React + TipTap 架构中还原飞书文档的用户可见效果：

- 文档内容在本地打开后，排版、间距、块样式、浮层位置尽量贴近飞书。
- 导入飞书公开链接、导出 HTML / ZIP 后，尽量保留原文档结构和视觉语义。
- 普通文档块、表格、多维表格、目录、评论侧栏分别有稳定的布局规范。
- 所有新增 UI 都遵循统一的颜色、字号、间距、圆角、阴影和 z-index 规则。

## 页面布局

### 文档页

- 顶栏固定在页面顶部，不参与正文滚动。
- 正文区域由 `doc-page-workspace` 承担纵向滚动。
- 左侧目录使用 sticky 布局，滚动正文时目录保持吸顶。
- 右侧评论栏使用 sticky / track 定位，与正文块锚点对齐。
- 编辑器版心默认宽度保持 `860px`，不要因窗口变窄随意重排正文内容。

### 正文版心

- 标题、作者信息和正文块左缘应保持一致。
- 有多维表格块时，标题区和多维表格内容需要共享同一左边界。
- 宽块（表格、多维表格、仪表盘）允许在版心内横向扩展，但不能遮挡目录和评论栏。

## 基础视觉 Token

当前项目散落在多个 Less 文件中的 token 应逐步收敛。新增样式优先使用以下值：

| 类型 | 值 |
| --- | --- |
| 主色 | `#3370ff` |
| 主色 hover | `#2860e1` |
| 正文文字 | `#1f2329` |
| 次级文字 | `#646a73` |
| 占位文字 | `#8f959e` / `#bbbfc4` |
| 浅边框 | `#dee0e3` |
| 浅背景 hover | `#f5f6f7` |
| 激活浅蓝背景 | `rgba(51, 112, 255, 0.08)` |
| 小圆角 | `4px` |
| 中圆角 | `6px` / `8px` |
| 浮层阴影 | `0 8px 24px rgba(31, 35, 41, .14)` |

## 普通文档块

### 标题

- H1 用于文档内一级标题，不等同于页面标题输入框。
- 标题块必须有稳定 `blockId`，目录和折叠状态都依赖它。
- 标题折叠状态保存到 `collapsed_heading_ids`，刷新后必须恢复。
- 标题折叠按钮只在该标题后存在子内容时显示。

### 段落与列表

- 段落、列表项、任务列表项都必须具备稳定块 ID。
- 空块显示左侧 `+`，非空块显示块柄。
- 空块退格遵循飞书式降级：标题 / 引用 / 代码块先变普通段落，再次退格合并或删除。

### 代码、引用、高亮块

- 代码块使用 lowlight 进行语法高亮。
- 引用块保留左侧强调线和缩进。
- 高亮块作为本地 callout 语义处理，不能退化为普通段落。

### 媒体块

- 图片、视频、文件卡片都作为块级元素处理。
- 图片支持裁剪、上下文菜单、删除和复制链接。
- 上传资源统一走 `/api/uploads`，静态访问统一走 `/static`。

## 表格

表格应优先对齐飞书的块级表格体验，而不是浏览器默认 table：

- 表格外围必须有 overlay 层，用于轨道选择、加行加列、边界灰点和块菜单定位。
- 选中行列时使用浅蓝选区。
- 单元格背景、表头、合并 / 拆分、列宽需要持久化到 HTML。
- HTML / TSV / Excel 粘贴应尽量保留链接、换行、表头和背景色。
- 复杂 rowspan / colspan 下的行列拖拽如果无法安全重排，应禁用并给出提示，而不是破坏表格结构。

## 左侧目录

- 目录必须随正文滚动吸顶。
- 目录项来自非空标题和文档标题。
- 当前编辑焦点所在标题或章节应同步高亮。
- 目录项过长时单行省略，不撑宽左侧轨道。
- 长目录在目录内部滚动，不影响正文滚动。

## 评论侧栏

- 评论定位优先依赖 `blockId`，不能依赖不稳定文档位置。
- 块级评论应在右侧评论栏中与锚点块对齐。
- 评论线程、解决状态、引用文本和锚点丢失状态应在 UI 中可见。
- 评论侧栏打开时，宽块不能覆盖评论栏。

## 浮层与菜单

所有浮层必须遵守以下规则：

- 块内浮层如果 portal 到 `document.body`，样式必须使用全局 `--portal` 类名。
- 浮层定位应使用 `getBoundingClientRect()`，并在视口 / modal / 文档容器内 clamp。
- 不要依赖被 overflow 裁切的父容器显示菜单。
- z-index 分层建议：
  - 普通块工具：`100` 级
  - 文档菜单 / Slash：`1000` 级
  - 多维表格菜单：`10000+`
  - 记录卡片 modal：高于普通菜单
  - modal 内 portaled 子浮层：高于 modal 内容但低于全局通知

## 多维表格 Bitable

Bitable 是独立系统，不应只当作普通表格处理。

### 通用

- Bitable 块必须是 `localBitableBlock`，支持整体删除、拖拽、复制块链接。
- Viewbar、工具栏、内容区必须对齐同一左边界。
- 多维表格浮层要避免被块容器 overflow 裁切。

### Grid 视图

- 字段头、行号、单元格、添加字段、添加记录区域需要保持固定网格线。
- 单元格展开、字段菜单、筛选 / 分组 / 排序菜单应使用 portal 或可靠浮层定位。

### Gallery 视图

- 卡片间距、封面比例、字段展示顺序应稳定。
- Toolbar 和卡片区不应出现大空隙。

### Kanban 视图

- 列菜单和卡片菜单不能被列容器裁切。
- 列内滚动应只作用在卡片列表，不影响列头菜单。
- 删除顶部“新建任务”按钮后，应保留底部或列内添加入口。

### Record Modal

- 记录详情弹窗内的附件上传、日期选择、字段菜单等浮层必须限制在白色 modal 内。
- 附件上传面板锚定字段右侧 `+` 按钮，不能漂到灰色遮罩区域。

## 导入后显示验收

导入飞书文档后，至少验收：

- 标题、正文、图片、表格和链接可见。
- 目录能正常生成和跳转。
- 多维表格相关内容能以本地 Bitable 或降级块展示。
- 不支持的飞书私有块必须产生 warning，而不是静默丢失。
- 页面刷新后内容、折叠状态、评论和附件仍可恢复。

## 不做的事情

- 不直接复制飞书私有压缩代码。
- 不承诺 100% 还原飞书私有协议。
- 不在没有结构化数据的情况下伪装成完整导入。
- 不为了单张截图引入破坏全局布局的临时 CSS。
