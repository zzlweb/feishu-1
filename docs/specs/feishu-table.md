# 飞书富文本表格规格

本文档定义本项目中飞书富文本表格的高保真复刻标准。后续涉及表格 UI/交互的修改，必须优先参考本文档，不得自行脑补。

## 1. 目标

复刻飞书文档富文本表格的核心视觉与交互：

- 表格主体样式
- 顶部/左侧操作轨道
- 行列边界灰点
- 插入行/列加号
- 行/列选中
- 表格块菜单
- 横向滚动
- 列宽调整

## 2. DOM/交互分层

飞书表格至少包含以下交互层：

| 层级 | 区域 | 作用 | 是否响应点击 |
|---|---|---|---|
| Table Body | 表格内容区 | 编辑单元格内容、显示选区 | 是 |
| Top Rail | 表格上边缘灰色轨道 | 列选择 | 是 |
| Left Rail | 表格左边缘灰色轨道 | 行选择 | 是 |
| Boundary Dots | 行/列边界灰点 | 插入行/列入口 | 是 |
| Plus Button | hover 灰点后的蓝色加号 | 执行插入行/列 | 是 |
| Insert Line | hover 插入点后的蓝色参考线 | 仅展示插入位置 | 否 |
| Block Handle | 表格左上角块柄 | 打开表格块菜单 | 是 |

强约束：

- Top Rail / Left Rail 负责选择。
- Boundary Dots / Plus Button 负责插入。
- 两者不能混用。
- 点击选择区不得触发插入。
- 点击插入区不得触发行/列选择。

## 3. 默认态

### 表格主体

- 单元格背景：白色。
- 单元格边框：浅灰，建议 `#dee0e3`。
- 表格默认不应整体蓝底。
- 表格菜单打开不等于整表蓝底选中。

### 操作轨道

- 顶部轨道位于表格上边界附近。
- 左侧轨道位于表格左边界附近。
- 背景为浅灰，建议 `#f5f6f7`。
- 轨道不可出现浏览器滚动条。

### 灰点

- 灰点位于行/列边界线上。
- 默认是小灰点，建议直径 `4px`。
- 灰点代表“可插入行/列的边界”。
- 灰点不是选择按钮。

## 4. Hover 态

### Hover 表格

应显示：

- 表格块柄。
- 顶部灰色轨道。
- 左侧灰色轨道。
- 行/列边界灰点。

不应显示：

- 整表蓝色高亮。
- 轨道滚动条。
- 非插入点位置的蓝色加号。

### Hover 灰点/边界插入点

应显示：

- 灰点变为蓝色加号。
- 对应插入参考线：
  - 列插入：竖向蓝线。
  - 行插入：横向蓝线。

不应触发：

- 行/列选中。
- 表格块菜单。

## 5. Click 行/列选择

### 点击 Top Rail

触发：

- 选中对应整列。

效果：

- 当前列所有单元格进入浅蓝选中态。
- 顶部轨道对应列段变为深蓝，建议 `#3370ff`。
- 不插入列。
- 不显示插入加号。

### 点击 Left Rail

触发：

- 选中对应整行。

效果：

- 当前行所有单元格进入浅蓝选中态。
- 左侧轨道对应行段变为深蓝，建议 `#3370ff`。
- 不插入行。
- 不显示插入加号。

### 实现要求

- 应使用 ProseMirror/Tiptap 的真实表格选区，例如 `CellSelection` + `TableMap`。
- 不允许只通过 CSS 伪造选中。
- 点击选择区时必须阻止事件穿透：
  - `preventDefault()`
  - `stopPropagation()`

## 6. Click 插入行/列

### 点击列边界加号

触发：

- 在对应边界插入列。

不应触发：

- 列选择。
- 表格块菜单。

### 点击行边界加号

触发：

- 在对应边界插入行。

不应触发：

- 行选择。
- 表格块菜单。

### 实现要求

- 插入命中区域应小于选择轨道区域，只覆盖灰点/加号附近。
- 加号点击也必须阻止事件穿透：
  - `preventDefault()`
  - `stopPropagation()`

## 7. 选中视觉

### 列选中

- 顶部轨道对应列段：深蓝。
- 表格中对应列：浅蓝选区。
- 深蓝条必须与列宽对应，不应偏移到边界点。

### 行选中

- 左侧轨道对应行段：深蓝。
- 表格中对应行：浅蓝选区。
- 深蓝条必须与行高对应，不应偏移到边界点。

### 非选中态

- 不得保留蓝色整表高亮。
- 不得保留错误的插入参考线。

## 8. 滚动行为

- 表格内容可横向滚动。
- 操作轨道本身不可显示浏览器滚动条。
- 横向滚动时，顶部轨道和灰点应与可视表格列对齐。
- 表格超宽时，操作控件不得漂移到正文外侧。

## 9. 表格块菜单

表格块菜单必须按截图/用户要求实现。

默认菜单包含：

- 同步块
- 缩进
- 剪切
- 复制
- 删除
- 分享
- 保存为模板
- 复制链接
- 标题行
- 标题列
- 均分列宽
- 在下方添加

禁止：

- 未经用户明确要求，不得新增截图外菜单项。
- 不得把单元格编辑菜单混入表格块菜单。

## 10. 表格选区浮动工具栏

当选中单元格、整行或整列时，应显示飞书式选区浮动工具栏。该工具栏不同于表格块菜单，负责当前选区的内容格式与表格结构操作。

### 第一阶段能力

- 设置为标题列。
- 左侧/右侧插入列。
- 上方/下方插入行。
- 合并单元格。
- 拆分单元格。
- 单元格背景色。
- 删除当前列。
- 删除当前行。
- 保留原有文本格式能力：标题、对齐、加粗、删除线、斜体、下划线、链接、行内代码、文字/背景色。

### 行为要求

- 只有表格选区激活时显示表格专项按钮。
- 合并单元格仅在多单元格选区可用。
- 拆分单元格仅在存在 rowspan/colspan 的单元格可用。
- 单元格背景色必须写入表格单元格属性，不能只做视觉层覆盖。
- 删除行/列必须基于当前真实表格选区或光标所在单元格。

### 后续阶段

- 整行/整列拖拽重排（Reorder）。
- 冻结首行 / 前 N 行。
- 冻结首列 / 前 N 列。
- 列宽内容自适应。
- 更高保真的飞书图标与 tooltip。

## 11. 验收清单

### 默认态

- [ ] 表格单元格白底。
- [ ] 单元格边框浅灰。
- [ ] 无整表蓝色高亮。
- [ ] 无操作轨道滚动条。

### Hover 表格

- [ ] 显示表格块柄。
- [ ] 显示顶部灰色轨道。
- [ ] 显示左侧灰色轨道。
- [ ] 灰点位于行/列边界。

### Hover 灰点

- [ ] 灰点变为蓝色加号。
- [ ] 显示对应插入参考线。
- [ ] 不选中行/列。

### 点击顶部灰色轨道

- [ ] 选中整列。
- [ ] 顶部对应列段变深蓝。
- [ ] 对应列单元格浅蓝。
- [ ] 不插入列。
- [ ] 不显示错误插入线。

### 点击左侧灰色轨道

- [ ] 选中整行。
- [ ] 左侧对应行段变深蓝。
- [ ] 对应行单元格浅蓝。
- [ ] 不插入行。
- [ ] 不显示错误插入线。

### 点击加号

- [ ] 点击列加号只插入列。
- [ ] 点击行加号只插入行。
- [ ] 不触发行/列选择。

### 表格选区浮动工具栏

- [ ] 选中列时显示浮动工具栏。
- [ ] 选中行时显示浮动工具栏。
- [ ] 点击合并单元格能生成正确 colspan/rowspan。
- [ ] 点击拆分单元格能恢复普通单元格。
- [ ] 单元格背景色能写入并持久化。
- [ ] 插入/删除行列不破坏当前表格结构。

## 13. AI 修改要求

每次修改飞书表格前，AI 必须先说明：

- 本次要修的具体行为。
- 涉及哪个交互层。
- 选择区和插入区是否会冲突。
- 预计修改哪些文件。

每次修改后，AI 必须说明：

- 实际修改了哪些文件。
- 是否通过构建验证。
- 哪些效果需要用户视觉确认。
